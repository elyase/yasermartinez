So a tragedy happened, and after a blissful three months of coding in Ruby, I’m back to working with PHP.

Today, me and a coworker both ran into an issue where after installing PHPUnit, it would fail silently, just returning a blank prompt after typing phpunit into terminal. After opening up my php_error.log, I found that it was throwing this error:Â  

PHP Fatal error: require_once(): Failed opening required 'File/Iterator/Autoload.php' (include_path='.:/Applications/MAMP/bin/php/php5.3.6/lib/php') in /usr/local/share/pear/PHPUnit/Autoload.php on line 45

If you’ve ever used PEAR, you probably know that it’s an absolute abomination and would probably work better as a torture device than as a functional package manger. Well, it turns out that there are many PHPUnit packages floating around the internet, and I just hadn’t gotten the correct one.

My process for updating PEAR and installing the most recent (as of 7/22/2012!) PHPUnit is below. Hope this helps!

# Updates pear to latest version
sudo curl http://pear.php.net/go-pear.phar > go-pear.php
sudo php -q go-pear.php

# Uninstalls phpunit and reinstalls correct version with all dependencies
sudo pear uninstall phpunit
sudo pear install --alldeps --force channel://pear.phpunit.de/PHPUnit
